# top
## 기능 및 특징
+ 시스템의 CPU,메모리,스왑 사용량, 프로세스 리스트, 로드 등을 살시간으로 표시
+ 3초(기본)마다 화면 갱신(유닉스 리눅스 명령어 사전, 2010 : 5초)
+ 종료, 우선순위 변경 등의 프로세스 관리 가능
+ 화면에 출력되는 기본값 : 현재시간, 시스템 업데이트(1) 시간, 시스템에 로그인한 사용자 수, 지난 1분, 지난 5분, 지난 15분간의 시스템 평균 부하
+ 이 목록 아래에 프로세스 정보, CPU 상태, 메모리와 스왑 상태를 출력

## 기본 구문
top [옵션]

## 주요 옵션
+ -b : 배치모드로 정보를 출력한다. 실시간 상화 대화형모드로 정보를 화면에 일렬로 출력한다.
+ -d delay : 지정한 시간(delay 초)의 간격으로 정보를 업데이트하여 출력한다.
+ -i idle : 토글값이 off일 때, idle 프로세스나 좀비 프로세스 정보를 출력하지 않는다.
+ -n num : 지정한 시간(num)만큼 업데이트 정보를 출력한다.
+ -p pid : 지정한 프로세스 ID(pid)의 정보만을 출력한다.
+ -q : 시간의 간격 없이 계속하여 업데이트 정보를 출력한다.
+ -s : 몇 개의 대화식 명령을 비활성화한다(시큐어 모드).
+ -S : 누적된 정보를 출력한다(cumulative 모드).

# ps
## 기능 및 특징
+ 리눅스 시스템에서 ps 명령어는 현재 실행 중인 프로세스의 정보를 제공하는 도구
+ 프로세스의 상태, 사용 자원, 프로세스 ID 등

## 실행 방법
$ ps [옵션]

## 주요 옵션
+ -e : 시스템의 모든 프로세스를 표시한다.
+ -f : 전체 형식으로 프로세스 정보를 표시한다.
+ -u 사용자명 : 지정된 사용자의 프로세스를 표시한다.
+ -p 프로세스ID : 지정된 프로세스 ID의 프로세스를 표시한다.
+ -l : 긴 형식으로 프로세스를 표시한다.
+ -a : 터미널과 연관된 프로세스와 다른 사용자의 프로세스를 표시한다.
+ -x : 터미널과 연관되지 않은 프로세스도 포함하여 표시한다.

## 프로세스 상태 코드
+ R (Running): 프로세스가 실행 중이거나 실행을 위해 준비되어 있다.
+ S (Sleeping): 프로세스가 대기 상태에 있으며, 특정 이벤트나 조건이 발생할 때까지 활성화되지 않는다.
+ T (Stopped): 프로세스가 중지되었거나 트레이스/디버그 중이다.
+ Z (Zombie): 프로세스가 종료되었지만, 부모 프로세스에 의해 아직 회수되지 않은 상태이다.
+ D (Uninterruptible Sleep): 프로세스가 디스크 입출력과 같은 시스템 활동에 의해 대기 상태에 있으며, 인터럽트할 수 없는 상태이다.

# jobs
## 설명 및 특징
+ 작업이 중지된 상태, 백그라운드로 진행 중인 작업 상태, 변경되었지만 보고 되지 않은 상태 등을 표시

## 실행 방법
jobs [옵션] [jobID]

## 옵션
+ -l : 프로세스 그룹 ID를 state 필드 앞에 출력한다.
+ -n : 프로세스 그룹 중에 대표 프로세스 ID를 출력한다.
+ -p : 각 프로세스 ID에 대해 한 행씩 출력한다.
+ command : 지정한 명령어를 실행한다.


## 세션의 상태값
| 상태 | 설명 |
|---|----:|
| Running | 작업이 일시 중단되지 않았고 종료하지 않고 계속 진행 중임을 뜻 |
| Done | 작업이 완료되어 0을 반환하고 종료했음을 뜻 |
| Done (code) | 작업이 정상적으로 완료했으며, 0이 아닌 코드를 반환했음을 뜻 |
| Stopped | 작업이 일시 중단됨을 뜻 |
| Stopped (SIGTSTP) | SIGTSTP 신호가 작업을 일시 중단했음을 뜻 |
| Stopped (SIGSTOP) | SIGSTOP 신호가 일시 중단했음을 뜻 |
| Stopped (SIGTTIN) | SIGTTIN 신호가 작업을 일시 중단했음을 뜻 |
| Stopped (SIGTTOU) | SIGTTOU 신호가 작업을 일시 중단했음을 뜻 |

# kill
## 명령어 기능 및 특징
+ Linux 시스템에서 kill 명령어는 프로세스에 신호를 보내어 그 프로세스를 종료하거나 다시 시작하는 데 사용
+ 시스템 리소스를 과도하게 사용하거나 응답하지 않는 프로세스를 안전하게 종료하는 데 유용

## 기본 구문
$ kill [옵션] [프로세스ID]

## 주요 옵션
+ -l : 모든 시그널 목록을 표시한다.
+ -s : 지정한 시그널을 보낸다.
+ -9 : SIGKILL 시그널을 보내어 강제 종료한다.
+ -15 : SIGTERM 시그널을 보내어 정상 종료를 요청한다.

## 주요 시그널
| 시그널 | 설명 |
|---|----:|
| SIGHUP | 세션 종료 시그널 |
| SIGINT | 인터럽트 시그널 |
| SIGKILL | 강제 종료 시그널 |
| SIGTERM | 정상 종료 요청 시그널 |
| SIGSTOP | 프로세스 일시 정지 시그널 |
